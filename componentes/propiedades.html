    # Propiedades

    Unha propiedade é un atributo do JavaBean que afecta á súa aparencia ou á súa conduta. Por exemplo, un botón ten as seguintes propiedades: tamaño, posición, cor de fondo, cor de texto, habilitado ou non, etc.

    Un compoñente ten definido o seu estado a partir dun conxunto de propiedades. As propiedades son variables definidas por un nome e un tipo de datos e toman valores concretos. Nun JavaBean as propiedades son privadas e non se ven dende fóra da clase que implementa o compoñente.

    As propiedades dun Bean poden examinarse e modificarse mediante getters e setters:

    • get: le o valor da propiedade. public <TipoPropiedade> get<NomePropiedade>( ) Se a propiedade é booleana o método get impleméntase así: public boolean is<NomePropiedade>()

    • set: cambia o valor da propiedade. public void set<NomePropiedade>(<TipoPropiedade> valor) Se unha propiedade non inclúe o método set entón dise que é unha propiedade de só lectura.

    Un IDE que cumpra coas especificacións dos JavaBeans sabe como analizar un Bean e coñecer as súas propiedades, examinando todos os métodos e fixándose naqueles que empezan por get e set.

    Ademais, o IDE asocia un editor de propiedades específico apropiado para o tipo de dato da propiedade, que permitirá á persoa programadora modificalas facilmente no momento de deseño.

    # Modificar graficamente unha propiedade dun compoñente

    Unha vez instalado o compoñente no IDE, este debe ser capaz de identificar as súas propiedades simplemente detectando parellas de operacións get/set, mediante a capacidade denominada introspección.

    O IDE invoca os métodos get para mostrar o valor das propiedades. E cada vez que se cambia o valor dunha propiedade, chámase ao método set apropiado para actualizar dito valor.

    O IDE poderá editar automaticamente calquera propiedade dos tipos básicos ou dos tipos Color e Font. Para poder editar propiedades de tipos complexos haberá que crear un editor de propiedades personalizado.

    Un editor de propiedades é unha ferramenta para modificar o valor dun tipo de propiedade particular. Os editores de propiedades úsanse na ventá Propiedades, que é onde se determina o tipo da propiedade. Para cada tipo de propiedade búscase un editor apropiado e móstrase o seu valor actual.

    A creación dun editor de propiedades usando Java supón programar unha clase que implemente a interface PropertyEditor (opens new window), que proporciona métodos para especificar como se debe mostrar unha propiedade na folla de propiedades.

    Por defecto, a clase PropertyEditorSupport (opens new window), que implementa a interface PropertyEditor, proporciona os editores máis comunmente empregados, incluíndo os tipos básicos, Color e Font.
